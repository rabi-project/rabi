## Summary

One paragraph. What changes, for whom, and why now.

## Motivation

The problem this solves. Include the concrete scenario that fails or is impossible today.
If a design partner or deployment surfaced it, say which (or anonymize).

## Design

The actual change, precisely. For proto/schema changes, include the diff or the new
message/field definitions inline. State validation rules and defaults.

## Compatibility

- Does this break existing adapters, clients, or stored objects? How is migration handled?
- Which proto package version does it land in (`v1alpha1` additive / new version)?

## Alternatives considered

At least one serious alternative and why it lost.

## Conformance impact

Which conformance tests are added or changed. A normative change with no conformance
test is not done.

## Unresolved questions

What the author knows they don't know.
